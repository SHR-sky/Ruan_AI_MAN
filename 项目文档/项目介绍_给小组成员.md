# 项目介绍：景区导览服务AI数字人

## 我们参加什么比赛？

第十五届中国软件杯大赛 · A组赛题 **A5-景区导览服务AI数字人**  
出题企业：锐捷网络（苏州）有限公司

## 我们要做什么？

做一个 **AI数字人导游系统**，覆盖三种使用场景：

| 场景 | 载体 | 用户 |
|------|------|------|
| 🖥️ **游客交互端** | Web / 手机H5 | 游客在景区扫码打开，跟数字人对话、问路、听讲解 |
| 📊 **管理后台** | Web | 景区管理员上传知识库、配数字人形象、看数据大屏 |
| 📡 **ESP32 随身终端** | 自制硬件 | 游客拿小终端，GPS定位 + 按键问路 + OLED显示 |

## 技术栈速览

```
后端    Python FastAPI + LangChain + Qdrant (向量库) + PostgreSQL
前端    游客端: Vue3 + TypeScript + Vite
        管理后台: Vue3 + TypeScript + Vite (与游客端分离)
硬件    ESP32 + GPS(NEO-6M) + OLED(SSD1306) + 按键
部署    Docker Compose（前后端各一个容器 + DB + Redis + Qdrant）
```

## 核心流程

```
游客说话/打字 → ASR语音识别 → RAG检索知识库 → LLM生成回答
    → TTS语音合成 → 数字人口型表情同步 → 展示给游客
                                              ↑
ESP32 GPS定位 ── 上报位置 ── 触发附近景点提醒 ──┘
```

## 项目目录结构

```
E:\AI_MAN/
├── backend/          # FastAPI 后端（所有API + AI服务）
```

## 各成员关注点

- **后端/AI**：`backend/app/services/` 下的 RAG、LLM、ASR、TTS、数字人服务 — 逐步接入实际模型
- **前端(游客端)**：`frontend/` — 数字人展示 + 语音/文本对话 + 路线推荐
- **前端(管理后台)**：`admin/` — 知识库管理 + 数据大屏 + 数字人形象配置
- **硬件**：`esp32_sample/scenic_guide_esp32.ino` — ESP32 GPS上报 + 按键问答 + OLED显示

## 快速启动

```bash

pip install -r requirements.txt
uvicorn app.main:app --reload

# 游客端
cd frontend
npm install && npm run dev

# 管理后台
cd admin
npm install && npm run dev
```
