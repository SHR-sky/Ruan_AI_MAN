# 使用方法

## 项目启动

### 方式一：本地开发（推荐）

**后端**（默认 SQLite，无需额外装数据库）：
```bash
cd backend
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
uvicorn app.main:app --reload
```
启动后访问 http://localhost:8000/docs 可在线调试所有 API。

**游客端**：
```bash
cd frontend
npm install
npm run dev
```
启动后访问 http://localhost:5173

**管理后台**：
```bash
cd admin
npm install
npm run dev
```
启动后访问 http://localhost:5174

### 方式二：Docker 部署

```bash
docker-compose up -d
```

| 服务 | 地址 |
|------|------|
| 后端 API 文档 | http://localhost:8000/docs |
| 游客交互端 | http://localhost:5173 |
| 管理后台 | http://localhost:5174 |

## 管理后台操作说明

打开 http://localhost:5174，左侧导航栏有三个页面：

### 1. 数据大屏

展示景区运营数据：
- 今日/本周/累计服务人次
- 游客满意度趋势
- 热门问答 TOP10
- 游客关注点词云
- 实时对话监听

### 2. 知识库管理

上传和管理景区资料：

**文档上传**：
1. 点击"选择文件"，支持 `.pdf` / `.docx` / `.md` / `.txt`
2. 点击"上传"，系统自动解析并入库
3. 上传后可在列表中查看、删除

**FAQ 管理**：
1. 输入"问题"和"答案"
2. 点击"添加"录入常见问答
3. 列表中的 FAQ 可逐条删除

> FAQ 优先级高于文档检索，高频问题走精确匹配，准确率接近 100%。

### 3. 数字人配置

| 配置项 | 说明 |
|--------|------|
| 名称 | 数字人称呼，如"小导" |
| 性别 | 男/女 |
| 声音类型 | 默认 / 温柔 / 活泼 / 专业 |
| 形象 | 上传图片或视频作为数字人外观 |

## ESP32 终端使用说明

### 硬件接线

| 组件 | ESP32 引脚 |
|------|-----------|
| GPS TX | GPIO 16 |
| GPS RX | GPIO 17 |
| OLED SDA | GPIO 21 |
| OLED SCL | GPIO 22 |
| 按键 | GPIO 0（接 GND 触发） |

### 烧录步骤

1. 安装 Arduino IDE，配置 ESP32 开发板支持
2. 打开 `esp32_sample/scenic_guide_esp32.ino`
3. 修改文件开头的 WiFi 和服务器地址：

```cpp
const char* WIFI_SSID = "你的WiFi名称";
const char* WIFI_PASS = "你的WiFi密码";
const char* API_BASE = "http://服务器IP:8000/api/v1/device";
```

4. 选择开发板 "ESP32 Dev Module"，选择对应 COM 口
5. 点击上传

### 终端交互

| 操作 | 效果 |
|------|------|
| 开机自动连 WiFi | OLED 显示 "WiFi OK" |
| 自动上报 GPS | 附近有景点时 OLED 自动显示名称和距离 |
| 按一下按键 | 显示附近景点列表（编号 + 名称 + 距离） |
| 走近景点(<50米) | 自动弹出"您已到达XXX"提示 |

### API 调试

所有 ESP32 设备端 API 均可在后端 Swagger 文档中测试：
http://localhost:8000/docs → **ESP32设备** 分组

常用调试接口：

| 接口 | 测试方法 |
|------|----------|
| 模拟 GPS 上报 | POST `/api/v1/device/gps`，填 `lat`/`lng` 参数 |
| 模拟提问 | POST `/api/v1/device/query`，填 `query` 和可选 `lat`/`lng` |
| 查附近景点 | GET `/api/v1/device/pois?lat=31.23&lng=121.47&radius=500` |

## 常见问题

**Q: 后端启动报错 "No module named 'aiosqlite'"？**  
A: 依赖没装全，运行 `pip install -r requirements.txt`

**Q: 前端请求后端报 404？**  
A: 确保后端先启动（端口 8000），vite 配置了代理会自动转发 `/api` 请求

**Q: Docker 部署时 PostgreSQL 一直重启？**  
A: 检查 5432 端口是否被占用，或先 `docker-compose down -v` 清掉旧卷

**Q: 怎么换大模型？**  
A: 修改 `backend/.env` 中的 `LLM_BASE_URL`、`LLM_MODEL`、`LLM_API_KEY`
